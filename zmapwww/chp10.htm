Chapter 8 Maps of z-values

Producing a Grid

ZMAP features some standard techniques, like selecting earthquakes, plotting cumulative number curves and magnitude signatures. The original idea that motivated the entire package is to provide an unbiased analysis of the seismicity over space and time.

For a spacial analysis of the seismicity rate changes using ZMAP, it is first necessary to calculate the cumulative number curve at each grid point. These grids can be saved and re-loaded for various analyses later on. Producing a grid is the most time consuming process in the analysis, as it takes up to one hour to calculate a large grid of 100x100 grid-points on a Pentium PC or Unix Sparc 20. So get a cup of coffee and hang on.

When you select "Create a Grid" from the "Tools" pulldown menu of the map window (Figure 19), an input box pops up. You will have to set the number of earthquakes per gridpoint and the latitudinal and longitudinal distance between grid points (grid spacing) in degrees. The number of earthquakes in each volume around a grid-point is critical for the resolution of the z-value maps. There is a trade off between the significance of the calculated z-values and the resolution. Small volumes, meaning few earthquakes, result in less significant z-values but a good spatial resolution, and vice versa. The volumes have to be overlapping to get a continuous image. For a more detailed discussion about the design of a grid see Wiemer and Wyss (1994). As an example, Figure 57 displays the grid we used to investigate the Landers region. For some selected grid-points, the circle containing 100 earthquakes are plotted, and you can see how the size of the circles is inversely proportional to the density of earthquake. At each grid-point the N clostest earthquakes are sampled, even when they are not within the boundary of the grid. To avoid sampling areas outside the grid, apply a pre-selection by polygon.

FIGURE 58. Input parameters to create a grid

A grid spacing that is too small, therefore a grid with too many gridpoints, can not only take a long time to be calculated but also crash the program! This happened several times using a 386. One option is to re-configure your PC so that other application programs (Word, File manager) do not occupy the extended memory.

As a rule of thumb, I successfully used ni = 100 and a grid with up to 70x70 points (preferably less) on a UNIX sparc20. Using a PC with 8 MB of internal memory I advise to use as a maximum a grid size of 35x35.

Click on "ZMAPGRID" in the makegrid input window when you have finished your selection of the grid parameters. You will be asked to interactively select the size of the grid on the map. Mark the lower left corner by hitting the left mouse button, then mark the upper right corner by hitting the left mouse button again. The outline of the grid will be displayed in the map.

A cumulative number curve is now calculated for each grid-point. A small window displays the percentage of the total calculation done. Once the gridding is completed, you can enter a filename where the grid will be saved. If you would like to use this grid later on again, load this grid file using the "Load *.mat Datafile" option in the main menu window. The grid, as well as the catalog used to calculate this grid, will be loaded. In a later session you can instead of loading the catalog and the grid separately load the grid-data-file from the "Main Menu window". Since the grid-data-file contains the catalog information as well, the catalog will be plotted in the Map window and you can open the menu of the different z-value maps from the tools menu of the map window. This shortcut also guaranties that you work with the same set of earthquakes as you used to produce the grid in the first place.

Once you have created a grid, you can apply various statistical functions to investigate changes in the seismicity rate. A description of these statistics is given by Haberman (1983,1987,1991). A section Menu will pop up automatically once the grid is produced. You can also open this menu from the tools menu of the map window. In the following I will first discuss the different statistics used and then the various presentation options.

The AS(t) Function

The AS(t) function (Habermann, 1983, 1987, 1991) was designed to answer a very specific question: Where in time is the strongest rate change between two specific end-points? In the past we have found the AS(t) function useful for the identification of an anomaly at the end of a time series, and for the identification of its onset-time. However, AS(t) may not detect anomalies when two or more rate changes occur in a time series. The z-value at time t at each grid point was calculated using the following equation (Habermann, 1983):

R1 is the mean rate in period 1 (from t0 to t), R2 the mean rate in period 2 (from t to te), where t is the `current' time (t0 < t < te). 1 and 2 are the standard deviations in these periods, and n1 and n2 the number of samples.

The LTA Function

For an unbiased statistical test of the significance of an anomaly, as well as to detect a period of reduced rate within a time series, the LTA-function (Long Term Average: Habermann, 1988, 1991) is more applicable:

The LTA function compares the overall mean rate Rall to the mean rate Rwl in a window of length wl (n indicates t he number of samples, 1 and 2 the standard deviation). This window is moved over the entire period in, for example, 2 week steps. The length of the LTA window introduces another parameter into the analysis.

The Rubberband Function

This function uses the same equation as AS, but different periods. It compares the seismicity rate in the window t0 to t to the seismicity in the window t to t+iwl. The window length iwl can be edited in the same way as described for AS. The advantage of this function is that it does not use "future" seismicity in this comparison and therefore models a real-time approach.

Percent

The percent comparison simply calculates change in rate 1 (to to t) and rate 2 (t to te) in percent. The range of this function is from -100% (no earthquakes in second period) to infinity (no earthquakes in first period).

Plotting a Map

After producing a grid, we have a time-series of z-values at each grid-point. This is a three-dimensional array of data, with a z-value at each point (x,y,t). One way to visualize this array is by slicing it in time. Therefore, we plot a latitude/longitude map of z-values at a specific time t. Each z-value is assigned a color. By viewing a sequence of time slices as an animation one can visualize seismicity rate changes in space and time.

Understanding Z-value Maps

Figure 61 shows an example of a z-value map for the central Hawaiian region prior to the 1983 Kaoiki earthquake. A few remarks for the understanding of z-value maps:

    • High (positive) z-values (hot colors in this presentation) represent a decrease in the seismicity rate. Low (negative) z-values represent an increase.
    • The maps are scaled relative to the maximum value, so one can expect a hot-spot in every map, but not every hot-spot has to be significant. You can change the scaling of the z-axis by clicking on "FixAx".
    • The location of a hot-spot is only the center of the volume that is actually sampled! In the example case, a large region is not covered by the catalog. In this situation you observe a 'smearing out' of the colors towards the edges due to the leaking of information from the area with data into areas without data. To avoid over interpretation an option was build into the program to select a maximum radius acceptable for N events. It is recommended to reduce this number until colors on the z-value map appear only on areas containing earthquakes.
    • The density and distribution of earthquakes is a critical factor for the understanding of a z-value map. Large areas with a constant color may just sample the same population of earthquakes over and over again - or may indicate a homogeneous seismicity rate over this area.
    • The seismicity is sampled in circular volumes (actually cylinders with no depth constraint). A linear seismicity feature, for example, may not show up as a linear feature in the z-value maps. Using "Circle" allows to view the distribution of the seismicity sampled. In the example shown in Figure 61, one can see a red spot in the upper right corner of the plot with a z-value up to 7. The best way to proceed from here is to use "Circle", and display the cumulative number curve at this point. The cumulative number window is identical to the display discussed earlier, various tools are now available for further analyses.

      The events selected by "Circle" are highlighted in the z-value map, just like they are in the map window. This allows the user to judge the sampled area of each circle. You can also use "Polygon" to select earthquakes within a polygon. Again, the cumulative number curve will be displayed.

Maximum Z-value Maps

 A more concise way to present the information of a three-dimensional array is to compress the array in one dimension. Sometimes one is mainly interested in the most significant rate change over the entire length of the catalog. Using the maximum value of the time series z(xi,yi,t) produces a map of maximum z-values that highlights the most significant changes in the entire catalog for a certain in LTA window length. The time information is lost, but can be reconstructed. Two neighboring points can represent a z-value at largely different times. This can be confusing, but in many cases we found the maximum z-value maps the most comprehensive tool to investigate seismicity rate changes and judge their relative significance. An example of a maximum z-value map, in the Landers region (California), is shown on the title page of this manual. The window length is 1.5 years, the hot spots indicate seismicity rate decrease prior to the 1992 Landers and Big Bear earthquakes (see Wiemer and Wyss, 1994).

Animating Z-value Maps

Instead of viewing just one time slice, the user can produce an animation or movie of time slices. In order to do that, several time slices are calculated, displayed, and stored as a bitmap. The advantage of this techniques is that these bit-maps can be displayed without the time consuming calculation of the z-values.

To produce an animation, select one of the animations in the z-value map menu (Figure 59). You will be asked for the number of slices, and, if you selected LTA, the window length of the LTA function. As a first step, ZMAP finds the maximum and minimum of all time slices, this is necessary to scale the images in a uniform way. Then ZMAP displays the slices on the screen and stores the bitmap.

The larger the displayed map on the screen, the larger the stored matrix of bitmaps. The size of a movie file can easily reach several MB. ZMAP will now display the first movie-frame (Figure 62). You can play the whole animation by selecting "Play". The number of runs and the speed in frames per second can be selected. Probability Maps

Probability maps are a first attempt to quantify the enhanced probability of a large earthquake occurring based on the seismic quiescence hypothesis. So far, we use a empirical relation based on earlier case studies. The probability is calculated based on the z-value using the equation (Wyss and Wiemer, 1995; Wiemer and Wyss, 1995)

z-values below 2.57 (99% confidence level) are considered normal and the probability of an earthquake occurring is equal to the Poisson probability (here judged for California and weighted with the average size of the anomaly). To modify this equation, edit line 18 and 19 in file src\view_max.m.

When you select "Probability" from the z-value map menu, the current z-value map will be transformed into a probability map. If nothing else, this transformation enhances the maxima in the data. Hit the "Back to z-value map" button in the tools menu to go back to the z-value map.

Resolution Maps

One finds that the z-value maps leak information into areas with no data. Large neighboring circles sample virtually the same population of earthquakes and therefore show the same color. Z-value maps tend to'smear out' towards the edges. To suppress this effect you can set the threshold value minimum radius in kilometers in the upper right corner of the z-value map window. That is, areas where the sampling circle is larger than the minimum radius will not be plotted. To visualize the density of earthquakes in the catalog you can choose "Resolution Map" from the Tools menu of the z-value map window. This will display a map where the color code of the overlying image is proportional to the radius of each circle.

The Alarm Cube

The alarm cube is a first attempt to visualize anomalous seismicity rate decreases in time and space in one figure. Z-value maps for all times are calculated using either the LTA or the Rubberband function. This can be a time consuming process, a waitbar indicates the percentage of the calculation done. A histogram of all these z-values is plotted first, then the alarm cube is displayed. An alarm is issued when a z-value exceeds a threshold that can be set in the lower right corner of the alarm cube window. The minimum possible threshold is z = 2.57. Each alarm is plotted as a red circle in the space-time cube. A blue line of Tw length is plotted as a tail to each alarm (where Tw is the window length of the LTA or Rubberband function in years).

FIGURE 68. Alarm cube input parameters

Once the cube is displayed, you can change the viewpoint using the viewer window (Figure 70). It is sometimes difficult to identify the space-time position of alarms, so rotating the cube in extreme positions (vertical or horizontal) helps to identify time or location. The alarms shown in the cube are also plotted in the seismicity map as circles.

You can eliminate the plotting of z-values in areas of low seismicity rate by setting the value of MinRad in the upper right corner. This will suppress coloring areas with a circle size larger than MinRad. You can also create an animation of the cube by selecting "Make Movie" from the lower left corner of the cube window. This will create an animation of the cube going through several different viewpoints. This movie uses lots of memory and might crash Matlab! Reduce the window size of the cube before you create a movie, this reduces the memory used.

Tools from the Alarm cube window

Find Anomaly Groups

The alarms in a typical alarm cube can usually be distinguished into a number of distinct groups of alarms. The distinction as made based on the separation in time and space. The "Find Anomaly Groups" option will identify distinct anomaly groups based on the following criteria:

    • Overlap in space: Two alarms belong to the same alarm group when they sample radius overlap.
    • Overlap in time: Two alarms belong to the same alarm group when the later alarm is declared while the first is still in progress. Both of these criteria need to be fulfilled. Alarm group will be displayed as differently colored circles (randomly assigned colors) in the seismicity map window, and the number of alarms found will be displayed in the matlab command window.

      Display one anomaly group

      This option will display the combined seismicity of one anomaly group. The seismicity from all individual alarms in each alarm group will be combined, and identical events will removed. The cumulative number window will display the time series of seismicity in the selected alarm group, and the location of the events will be marked in the seismicity map.

z-values in cross-sections

 In many instances we found that cross sections along (or perpendicular to) the strike of faults reveal information indiscernible in map views (Wyss and Wiemer, 1996). For example, a seismicity rate decrease at depth can be obscured by a seismicity rate increase for shallow earthquakes, thus resulting in a map view indicating little change. In a cross section the difference between the two volumes can be mapped. Since many large earthquakes initiate in the lower part of the seismogenic crust, seismic quiescence may be limited to deeper volumes. We have expanded our initial z-value map view to cross-sections of z-values, using the same gridding technique described for z-value maps. After creating a cross-section (See "Cross-Sections" on page 42.) you van select the 'Make Grid' option from the Cross-Section window (Figure 27). An input window basically identical to that shown in Figure 58 will appear, only that you now select the grid spacing in distance and depth. The units are kilometer. After clicking the'Go' button, you need to define with a mouse-click the lower left and upper right corner of the grid in the cross-section plot. Once the calculation is completed, a menu will pop up, giving you the choice of time-cuts using various functions (LTA, AS, Percent, Rubberband). The z-value maps are displayed in as described above.
