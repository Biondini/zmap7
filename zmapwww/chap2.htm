Chapter 2 Getting Started

General Remarks

 This User Guide describes most of the features of ZMAP. The Users Guide is written more as a manual or reference guide and only briefly describes the reasons for developing the tools contained in ZMAP, and the applications we envision for these tools. For more information on how these tools can be used to address scientific questions, please refer to the published articles that use ZMAP (Wiemer and Wyss, 1994, Wiemer and Benoit, 1996; Wiemer and McNutt, 1996; Wiemer and Wyss, 1996; Wyss and Wiemer, 1996ab, Zuniga and Wiemer, 1995; Allmann, 1996). The figures of various ZMAP windows are screen-dumps from the UNIX version. However, the windows in the PC and Mac version are basically the same as in the UNIX version, only the shape of buttons and menus might be slightly different.

System Requirements

PC systems

In order to run ZMAP you need to install Matlab on your PC. Contact MathWork at info@mathwork.com for general information and prices. Please read the system requirements and installation part of the Matlab Users Guide for further details about the system requirements. Essentially you need an Intel Pentium or 486 processor, at least 8 Mb of extended memory, and a 387 or 487 co-processor chip (except 486DX or Pentium). We tested ZMAP on a 468DX, 66 MHz Gateway PC and a Pentium 133 MHz. To be able to run all the features ZMAP offers you will need to have a 256 color display, otherwise z-value maps will look poor and movies cannot be created. Check the "Windows Setup" for a device driver that supports 256 color.

We loaded earthquake catalog of up to 5000 events without problems on an 8 MB memory machine. The plotting of the maps is a little bit slow for a large number of events.

Unix systems

Using a SUN Sparc 20 with 64 MB memory, we loaded catalogs of up to 100.000 earthquakes before we started to run out of internal memory. ZMAP should run under SOLARIS as well as SunsOS, as long as matlab is functional. The stress tensor inversion routines have been compiled under Solaris 2.5 on a SUN workstation, and have not been tested on other systems.

Macintosh

The Macintosh version has the one difference that only one mouse key can be used. To circumnavigate the few instances where two mouse-keys are needed, the keyboard can be used as a substitute. Selecting the `1' key for example will end the data entry for a polygon selection.

Installing ZMAP on Your Computer

PC floppy disk

Copy the two floppies with (1) the source code, and (2) the sample datasets on to your hard-disc. The directory structure of the ZMAP home directory is shown in Figure 6. The home directory 'zmap' contains the file 'README' and the script-file 'zmap.m'. The sub-directory 'eq_data' contains the earthquake datafiles as well as coastline files, faults etc. The sub-directory 'out' is used for output of data from ZMAP (e.g. gridfiles, cumulative number curves). The sub-directories 'src', 'src/utils', and 'src/declus' contain the Matlab script-files used for ZMAP. The directory myfiles is empty. Here you can add you own functions or keep modified versions of ZMAP script files. When you update ZMAP to a later release, this directory will not be overwritten. Also, the priority of the directory structure is such that if a file with identical name exists in src and myfiles, the file in your personal directory myfiles will be executed.

UNIX Version

Uncompress the files [uncompress *.Z], untar the files [tar xvf zmaphtml.tar, tar xvf zmap.tar, tar xvf manual.tar.Z]. This will create the subdirectories src, doc, eq_data, myfiles and out. Print the Users Guide located in the /doc directory. The files are postscript files created with FrameMaker. If you are not able to print the Users Guide I can mail you a hardcopy. Starting ZMAP

To start ZMAP you have to start Matlab first by clicking on the Matlab icon (PC) or typing Matlab (UNIX). Then type in the Matlab command window:

cd <path>\zmap (or cd <path>/zmap on the UNIX system) where <path> is the path to the zmap home-directory. Now type:

zmap This will start ZMAP and two windows will pop up on your screen: (1) The welcome window, and (2) the ZMAP main menu window. The Matlab command window is used occasionally to input data, display messages and errors. You should keep it on the screen, but you may change the size or location.

If you already have a datafile, or would like to use one of the example datafiles, click on "Load *.mat file and go". An input window will ask you for your selection. After you have selected a datafile, ZMAP will ask you for various input parameters (Figure 9). You can limit your analyses to a part of the catalog, limited in time and magnitude. You can adjust the magnitude threshold above which earthquakes in the map and in the cumulative number curves are displayed with an 'x'. Earthquakes are displayed in different colors according to their depth. Click on the "Go" button after you complete your selection. This will pop up a map of the seismicity on your screen (Figure 19).

Help in ZMAP

On-line help is available in many windows. The "Info" button in each individual window will display a help text (Figure 10 ). Help about a specific function can be obtained by typing 'help function-name'. This is true for general Matlab functions as well as (some) ZMAP functions. Unfortunately, in some cases the `Info' button may display help associated with an earlier window.

FIGURE 10. Example window of the on-line help option

Printing in ZMAP

To print a figure in ZMAP, hit the "Print" button in the upper left corner of most windows. A new window, the print dialog box, will open (Figure 11). You can print to the default printer installed under Matlab by selecting "OK", or you can print the figure into a file. The print will not contain menus and buttons. To print a color figure make sure to select the `psc' or psc2' option. If you want to change the paper size or plot orientation, use the PageSetup option (Figure 20). For more information about printing in Matlab and the different device drivers type 'help print' and 'help printopt' in the Matlab command window.

We have imported ZMAP figures in a number of programs, and found that for the UNIX system post-script files work well with FrameMaker when converted to epsi files (uses ps2epsi, part of the ghostscript language). IslandDraw work well for most figures, but does not handle interpolated colormaps (or sometimes not-interpolated colormap) well. However, the advantage is that the postscript-file can be ungrouped and individual elemts (such as font-sizes, line-thickness) modified after the figure has been completed. ON PC systems, I have succesfully imported windows metafiles from Matlab into Microsoft Powerpoint 7. Use the print -dmeta option to print your figure. Again, individual elemts of the figure can be post-processed, but I have not been able to import interpolated colormaps.
