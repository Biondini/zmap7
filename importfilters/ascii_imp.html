ASCII file format: to import a catalog from an ascii file, have your data as columns in one of the common delimited formats (tab, comma, '|', spaces, etc.) The first row should be a header that is similarly delimited.

Columns recognized by Zmap:

Column order does not matter, and additional columns may exist that will not be interpreted.

Required Columns

  • Date recommended formatted as YYYY-MM-DDThh:mm:ss with seconds expressed with up to 9 decimal places. However all dates must be formatted the same
  • Longitude (decimal)
  • Latitude (decimal)
  • Depth
  • Magnitude

Optional Columns

  • MagnitudeType , ex. ml
  • Dip
  • DipDirection
  • Rake

Alternate fields

Dates can alternatively be expressed with a combination of fields:
  • year
  • month
  • day
  • hour
  • minute
  • second
  • decyear
  • time, as hh:mm:ss

Beware: The importer might be confused by an input file that contains multiple columns with similar names.

Examples

Example 1. pipe '|' delimited data 
year|month|day|hour|minute|lat|lon|dep[m]|mag|magtype
2001|03|12|12|01|42.3|-120.5|20.0|4000|Ml
2001|05|19|01|00|42.4|-120.2|15.0|3200|Ml
2001|12|15|13|32|42.3|-120.3|13.0|2900|Ml

Example 2. Comma delimited data, with a column that will be ignored 
Latitude,Longitude,Date,depth,magnitude, something
42.3,-120.5,2001-03-12T12:01:32,20.0,4,abc
42.4,-120.2,2001-05-19T01:00:00,15.0,3.2,def
42.3,-120.3,2001-12-15T13:32:05,13.0,2.9,ghi

Example 3. Tab-delimited data 
year	month	day	Time	latitude	longitude	depth	magnitude
2001	03	12	12:01:32	42.3	-120.5	20.0	4
2001	05	19	01:00:00	42.4	-120.2	15.0	3.2
2001	12	15	13:32:05	42.3	-120.3	13.0	2.9
