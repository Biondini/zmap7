The probability that a rate change is significant

 

Rate changes are commonly measure by z- or beta values. However, the statistical values do not immediately translate into significance, because significance is a function of sample size, window length, and binning size.  For each data-set, the significance threshold must therefore be determined based on a random simulation. There are two options to perform such a sumilation:

 

1) Compared to a ‘perfect’ stationary poissonian time series. This will generally result in high significances.

 

2) Compared to randomly drawn samples from the real data. This will result in lower significances, since the overall imperfections the data are inherently considered.

 

It is not quite clear which option I the right one to choose.

 

 

 

After plotting a time series and z- or beta value curve, using an LTA approach, you can compute a translation into probability using the ‘Translate into probability’ button. You fisrt need to decide if you like option 1 (uniform) or 2 (real data) for the simulation. Next the number of random samples created for comparison must be defined. Finally, a plot will display the results.

 

 

Show is the significance of rate decreases (in red) and rate increases (in blue). Colors are only displayed when the rate changes is at least 5% significant.
