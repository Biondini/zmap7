First steps – installing and starting ZMAP

 

1) Requirements

ZMAP consists numerous Matlab scripts and functions (m-files).  There are also a few FORTRAN or C codes that are called for specific tasks (e.g., stress tensor inversions). The core of ZMAP should run on any system Matlab runs on. ZMAP has been tested under Windows PC (95, 98, 2000, NT), HP UNIX 10.x and 11.x, SUN SOLARIS and LINUX. We recommend a recent version of Matlab (5.3, 6.x). Specific toolboxes are not required, but some functionalities of ZMAP will only be available if the mapping toolbox or statistic toolboxes are installed. Some applications can be quite memory intensive (large datasets, topographic maps), and Matlab generally does not recover well once memory limitations are reached.

 

2) Installing ZMAP from the Internet

The most recent version of ZMAP can be obtained from http://seismo.ethz.ch/staff/stefan. You can download either a UNIX tar file, which would be the best option for HP, SUN, or LINIX, or a self extracting *.exe file for PC. These versions are identical except for the external calls to FORTRAN and C routines, which are explained later on.

 

On a UNIX system:

Uncompress the file (uncompress ZMAPv6.tar.Z), and untar it (tar –xvf ZMAPv6.tar). This will create a subdirectory ZMAP6.

 

On a Windows PC:

Double click the ZMAPv6.exe. This will open a WinZip self-extractor. The files will be installed in a directory called ZMAP6.  

 

3) Starting ZMAP

Start Matlab on you system. Change into the ZMAP installation directory (e.g., cd c:\ZMAPv6) and type zmap . This should bring up the two initial windows.  Test out the basic functions by clicking on the ‘Load *.mat ‘ button from the menu. Load a file from the ./eq_data directory. An input selection window should appear. Press ‘Go’, and a map should appear. Go to Ztools and select the “Analyze time series …” button. The cumulative number will appear.  Finally, check the connection with your web browser by selecting the “Instruction & Help” button from the Menu. You browser should open, along with possibly the Matlab help browser. Both should display the Help web page. If not, open your browser manually and try again. If there is still no window, check out the Matlab help for the command ‘web’, and see if you can open any of the listed example web pages (e.g., web http://www.mathworks.com  -browser).

 

4) What’s next?

Check out the cookbook (in ./help/cookbook.pdf) and help options listed from the “Introduction & Help” section. A number of precompiled datasets are available in the eq_data directory, including a two (Harward and Southern California) with focal mechanism information. Download additional datasets available from our database. Play around with some of the functionalities to familiarize yourself with ZMAP.

 

5) External calls in ZMAP

If you are planning to run stress tensor inversions, you need to run several executables. These maybe already precompiled for your operating system.

 

On a PC: The necessary *.exe files are included in the ZMAPv6.exe distribution and should run without problems.

Under UNIX: Several makefiles have been provided by Andy Michael as part of his stress tensor inversion software. The can be found in the ./external directory. Specifically, you will need to run:

 

makeslfast

makeslick

makebtswl

 

Depending on your system, you may have to change some compiler options.

 

Gephart’s stress tensor inversion program and the cumulative misfit method are precompiled for the Windows version. If you want to run them on a different operating system, please contact me, I will provide you with the necessary codes.

 

If you would like to use Reasenberg’s declustering code as a FORTRAN executables rather then the Matlab internal version (because it is faster and you do get the output of the clusters in Reasenbergs form), you need to compile the file ./external/myclus.f. (e.g., f90 –o myclus myclus.f) This should also already work in the PC version.
