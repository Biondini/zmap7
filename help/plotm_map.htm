Plotting maps using m_map

 

A useful tool, called m_map (http://www2.ocgy.ubc.ca/~rich/map.html)  to plot nice maps was written by Rich Pawlowicz (rich@ocgy.ubc.ca). ZMAP uses m_map to produce reasonable looking maps when the standard Cartesian coordinate system used in Matlab just doesn’t cut it. The m_map code is distributed as part of ZMAP in the m_map and m_map/private subdirectories.

 

From the Map-window in ZMAP. choose the ZTOOLS -> Plot map using _map/Import coastline option,. This will open an input box:

 

 

The first line allows you to choose various projections. If you are familiar with m_map and you would like to add new projections, you will need to edit the file m_map/plotmymap.m. Next choose a suitable resolution suitable for your map. If you choose too high a resolution, the map will take a long time. As part of the ftp distribution of ZMAP, the crude and low GSSHS files are included in the  m_map/private directory. If your resolution is not available, an message will remind you to download the appropriate file from ftp://ftp.ngdc.noaa.gov/MGG/shorelines/

 

The map will then be plotted in a separate window. Use the button in the lower right to make transfer the plotted coastline into the ZMAP map window. If you permanently want to attach this coastline with the earthquake data set, choose now from the mapping window the Catalog -> Save current catalog as mat file   option.
